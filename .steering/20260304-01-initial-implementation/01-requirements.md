# 要求内容 - 初回実装

## 概要

Growi Wikiのリビジョン差分確認プラグイン（growi-plugin-revision-diff）を新規に開発する。

## 変更・追加する機能の説明

### 1. Sidebarボタン

- Growi Sidebarに「差分確認」ボタンを追加する
- [growi-plugin-all-seen-users](https://github.com/jajamaru09/growi-plugin-all-seen-users) と同じ方式で実装
  - Growi標準Sidebarの既存ボタンと同一のHTML構造・CSSクラスを動的生成して注入

### 2. 差分確認モーダル

- ボタンクリックで差分確認モーダルを開く
- 左右2カラムレイアウト
- 各カラム上部にリビジョン選択ドロップダウン
  - 表示形式：`{revisionNo} - {revisionId} - {更新日時}`
  - revisionNo：更新日時昇順でソートし、古いものから1始まりの連番を付与
- 2つの表示モード（Markdown / HTML）の切り替えタブ

### 3. Markdown差分モード

- 左右のリビジョンのMarkdownテキストを並べて差分表示
- 追加行：淡い緑（`#e0ffe0`）でハイライト
- 削除行：淡いピンク（`#ffe0e0`）でハイライト

### 4. HTML差分モード

- 左右のリビジョンのMarkdownをGrowiレンダリングエンジンでHTMLに変換
- レンダリング済みHTMLに対して差分箇所を検出
- 追加部分：淡い緑（`#e0ffe0`）でハイライト
- 削除部分：淡いピンク（`#ffe0e0`）でハイライト

## ユーザーストーリー

- Wikiユーザーとして、閲覧中の記事のSidebarに表示された差分確認ボタンをクリックし、モーダルを開きたい
- Wikiユーザーとして、左右のドロップダウンで比較したい2リビジョンを選択し、差分を確認したい
- Wikiユーザーとして、Markdown形式の生テキストと、HTMLレンダリング済みの表示形式の両方で差分を確認したい

## 受け入れ条件

- [ ] Growi Sidebarにネイティブスタイルと一致したボタンが表示される
- [ ] ボタンクリックでモーダルが開く
- [ ] モーダル内のドロップダウンに `revisionNo - revisionId - 更新日時` 形式でリビジョン一覧が表示される
- [ ] revisionNoが更新日時昇順で1始まりの正しい連番で表示される
- [ ] Markdown差分モードで差分がハイライト表示される
- [ ] HTML差分モードでGrowiレンダリングを用いた差分がハイライト表示される
- [ ] 追加箇所は淡い緑、削除箇所は淡いピンクでハイライトされる

## 制約事項

- Growiのプラグインシステム仕様に準拠
- Growiコアのソースコードを改変しない
- HTMLレンダリングはGrowiエンジンを使用する（外部変換ライブラリのみの使用禁止）
- SidebarボタンはGrowiネイティブUIと同一スタイルを維持する
